# ReadMe

タイトル：The Circular Library (The Climbは旧名です)

ジャンル：2Dローグライクアクション

プラットフォーム：Windows11

プレイ時間目安：約1時間

制作期間：約７か月（木土日祝、一部日にちを除く　1日の平均作業時間：5時間）

制作環境：Unity 6000.2.6f2



〇ゲーム説明

本作は、6つの異なる2Dアクションステージを攻略し、それぞれで獲得したスキルを他ステージに持ち込んで活用する構造のゲームです。

各ステージは独立したゲーム性を持ちながら、取得したスキルによって多様な攻略ルートが存在します。これにより、プレイヤーの発想次第で効率的な攻略や、本来の想定とは異なるショートカットが可能となっています。



〇チーム情報

チーム人数：8人

役割構成：プログラマー5人、プランナー1人、プログラマープランナー兼用2人


自分の役割：主軸プログラマー


自分の担当箇所：

プレイヤーの挙動・『陰陽相棒ステージ』の全般



〇操作方法

　　　　　　　　移動：A/Dキー
        
　　　　　　ジャンプ：Spaceキー
      
　　　　　　　スキル：左クリック/Qキー
       
　　　スキル切り替え：Shiftキー
   
ステージ固有ギミック：右クリック/Eキー



〇アピールポイント


陰陽相棒ステージの設計と実装


内容：

パズル要素、専用敵、ボス戦を組み合わせたステージの設計および実装を担当しました。

また、スキルを活用することで本来の想定ルートを省略できる「ショートカット」が成立する構造を意図的に組み込んでいます。


苦労した点・改善した点：

・スキル使用による想定外の挙動（進行不能やゲーム進行の破綻）を防ぐための制御を行いました。

・ギミック、敵、スキル間の相互干渉を整理し、複数要素が同時に作用しても破綻しないよう調整しました。



〇補足


反省点と今後の改善点：

プレイヤーに関する実装において、開発途中で挙動の分離や状態管理の整理など複数の改善案が見つかりましたが、既存システムとの依存関係が強く、大規模な修正が困難であったため対応できませんでした。

この経験から、初期段階で処理の分離および状態管理を明確にし、後からの機能追加や仕様変更に対応しやすい構造を設計する必要があると認識しました。今後は、プレイヤー処理をステート単位で分割し、各機能の独立性を高めた設計を行います。

また、企画が一度白紙となり再設計が発生したことで、開発期間の多くを消費しました。

今後は、初期段階でプロトタイプによる検証を行い、大きな仕様変更の発生を防ぐ進行を意識します。

未実装要素：ラスボス、キーコンフィグ、個別の音量調整、ゲーム説明、チュートリアル
